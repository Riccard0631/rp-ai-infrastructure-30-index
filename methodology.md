# RP AI Infrastructure 30 Index Methodology

## Objective

The RP AI Infrastructure 30 Index (ticker: RPAI30) is an informational benchmark designed to track the price performance of publicly traded companies connected to artificial intelligence infrastructure. The index focuses on semiconductors, semiconductor equipment, cloud platforms, data centers, networking, power infrastructure, cybersecurity, and observability.

The index is not an ETF, fund, advisory service, regulated investment product, or tradable product.

## Weighting

The index is equally weighted. Each component receives a target weight of 1/30, or approximately 3.3333%, at each scheduled rebalance.

Daily index returns are calculated as the average of component price returns. When a component does not have a valid price return for a given date, the calculation uses the available component returns and re-normalizes weights across the available components.

## Rebalance Policy

The index rebalances quarterly. At each rebalance, all active constituents are reset to equal target weights.

The MVP implementation stores target weights in `components.csv`. A production implementation should add a dated rebalance calendar, divisor records, corporate action treatment, constituent change logs, quality-control reports, and data-vendor reconciliation.

## Calculation Methodology

The index is a price-return index.

- Index name: RP AI Infrastructure 30 Index
- Ticker: RPAI30
- Base value: 1000
- Base date: 2026-06-18
- Component count: 30
- Weighting: Equal weight
- Rebalance frequency: Quarterly

For each calculation date:

1. Download daily close prices through yfinance.
2. Calculate each component daily price return.
3. Calculate the equal-weight average component return.
4. Update the index level using:

```text
index_today = index_yesterday * (1 + average_component_return)
```

The MVP uses yfinance close prices for price-return calculations. Users should validate split, dividend, and corporate-action treatment before relying on the output for research.

The public dashboard also includes a historical simulation generated by `backtest.py`. The simulation is separate from the official live index history, which starts on the 2026-06-18 base date.
